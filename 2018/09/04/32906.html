部署虚拟环境安装 Linux 系统

RPM 与 Yum 软件仓库

RPM 能够完成依赖关系检查，为了解决源码安装的复杂依赖关系而诞生。

目前采用该方案的有：CentOS、Fedora、openSUSE

Yum 软件仓库可以根据用户的要求分析出所需软件包及其相关的依赖关系，然后自动从服务器下载软件包并安装到系统。

systemctl 命令

用以管理服务。

Tips: foo.service 中的 .service 可以省略。

序号命令说明
1systemctl start foo.service启动服务
2systemctl restart foo.service重启服务
3systemctl stop foo.service停止服务
4systemctl reload foo.service不终止服务的情况下重载配置
5systemctl start foo.service查看服务状态
6systemctl enable foo.service开机启动
7systemctl disable foo.service禁止开机启动
8systemctl is-enabled foo.service检查特定服务是否开机启动
9systemctl list-unit-files –type=service查看各个级别下服务的启动、禁用情况
